# archery_parser

**Ianseo PDF qualification result → structured CSV pipeline**

Converts archery competition result PDFs generated by the
[Ianseo](https://www.ianseo.net/) competition management system into
arithmetically-verified CSV files suitable for database import and
statistical analysis.

Developed for the Estonian archery competition circuit by
**Vana-Võidu Vibuklubi**.

---

## Installation

**Requirements:** Python 3.10 or newer.

```bash
# Clone the repository
git clone https://github.com/your-org/archery-parser.git
cd archery-parser

# Install (editable mode recommended for development)
pip install -e ".[dev]"
```

Dependencies installed automatically:
- `pdfplumber >= 0.10` — PDF text extraction with bounding-box coordinates
- `pytest >= 7` (dev) — test runner

## Usage

### Single file

```bash
python -m archery_parser input.pdf
# Output: input.csv (same directory, same stem)
```

### Multiple files — combined into one CSV

```bash
python -m archery_parser file1.pdf file2.pdf file3.pdf --output season.csv
```

### Append to an existing CSV

```bash
python -m archery_parser new_event.pdf --append results.csv
```

### Verbose logging to terminal

```bash
python -m archery_parser input.pdf --verbose
```

### Dry run (parse only, no output written)

```bash
python -m archery_parser input.pdf --dry-run --verbose
```

### Save log to file

```bash
python -m archery_parser input.pdf --log run.log
```

### Full argument reference

```
positional arguments:
  PDF                   One or more Ianseo qualification protocol PDF files

optional arguments:
  -o, --output CSV      Output CSV file path (default: <input>.csv)
  --append CSV          Append rows to existing CSV (no second header row)
  -v, --verbose         Print DEBUG-level log lines to stderr
  --dry-run             Parse only; do not write any output file
  --log FILE            Write log output to FILE as well as stderr
  --encoding ENCODING   Output CSV encoding (default: utf-8)
```

## Output format

Each athlete record is expanded into multiple CSV rows — one per
individual end score, one per half-subtotal, and one grand-total row.

| Column      | Type    | Example              | Notes                              |
|-------------|---------|----------------------|------------------------------------|
| Date        | string  | 14.09.2025           | End date of competition (DD.MM.YYYY) |
| Athlete     | string  | Martin Rist          | Firstname Lastname (title case)    |
| Club        | string  | VVVK                 | Code + name if both available      |
| Bow Type    | enum    | Recurve              | Recurve · Compound · Barebow · Longbow |
| Age Class   | enum    | Adult                | Adult · U21 · U18 · U15 · U13 · U10 · +50 |
| Gender      | enum    | Men                  | Men · Women                        |
| Distance    | string  | 70m / 2x70m / 4x70m | Encodes which score this row holds |
| Result      | integer | 318                  | Score for this Distance label      |
| Competition | string  | Puiatu CUP 2025      | Identical for all rows in the file |

## Architecture

The parser runs as five sequential, independently-testable pipeline stages:

```
PDF file(s)
    │
    ▼
Module 1: reader.py      pdfplumber → Word objects with (x,y) coords
    │
    ▼
Module 2: detector.py    Identifies section boundaries, column headers
    │                    → CompetitionMeta + list[RawSection]
    ▼
Module 3: assembler.py   Groups 1–4 printed lines per athlete
    │                    → list[AthleteRecord]
    ▼
Module 4: transformer.py Name/club/lookup transforms + row expansion
    │                    → list[CSVRow] (arithmetically verified)
    ▼
Module 5: writer.py      Writes UTF-8 CSV with header row
    │
    ▼
output.csv  +  run.log
```

## Running the tests

```bash
# All unit tests (no PDF required)
python -m unittest discover -s tests -v

# With pytest (if installed)
pytest tests/ -v
```

### Integration test

The integration test requires the real **Puiatu CUP 2025** PDF fixture:

```bash
cp /path/to/Puiatu-CUP-2025.pdf tests/fixtures/
python -m unittest tests.test_integration -v
```

Without the fixture the integration tests are **skipped** (not failed),
so `python -m unittest discover` always exits 0 for a clean install.

## Known limitations (v1.0)

- **Qualification rounds only** — match play / elimination bracket PDFs
  are not supported.
- **Team rounds** not supported.
- **Non-Ianseo PDFs** will fail to parse (competition header not found).
- **CLI only** — no GUI or web interface.
- **CSV output only** — no direct database insertion.
- Tested against Ianseo-generated PDFs from Estonian competition circuit;
  other country configurations may differ in section-title wording.
- 216-arrow and 288-arrow formats are supported in the data model but
  have not been tested against real PDFs.
